# Context Check Skill

사용자가 요청했을 때 Claude Code 세션의 컨텍스트 상태를 진단하고, 필요 시 새 세션 시작을 권고한다.

---

## 1. 컨텍스트 사용량 추정

API 메타데이터 없이 텍스트 길이만으로 토큰을 추정한다.

### 토큰 추정 규칙

| 언어/유형 | 비율 |
|-----------|------|
| 영어 텍스트 | ~4자 = 1 토큰 |
| 한국어 텍스트 | ~1.5~2자 = 1 토큰 |
| 코드 | ~3~5자 = 1 토큰 |

현재 모델(claude-sonnet-4-x / claude-opus-4-x)의 최대 컨텍스트: **200,000 토큰**

현재 대화의 전체 텍스트 길이를 기반으로 추정 토큰 수를 계산한다:
```
추정 사용량 (%) = (추정 토큰 수 / 200,000) × 100
```

### 경고 신호 패턴 감지

아래 항목을 체크해 보정 근거로 활용한다:

| 신호 | 가중치 |
|------|--------|
| 대화 turns이 30회 이상 | +10% 보정 |
| 큰 파일(1000줄+)을 3회 이상 읽음 | +15% 보정 |
| 대형 diff/코드 블록이 다수 존재 | +10% 보정 |
| 이전 내용을 Claude가 "잊어버린" 듯한 패턴 | +10% 보정 |
| 에러: context length exceeded | 즉각 위험 판정 |

### 파일/대화 히스토리 분석

- 현재 대화에서 읽어들인 파일의 수와 총 라인 수를 파악한다
- 반복적으로 로드된 파일이 있으면 이를 명시한다
- 대화 흐름에서 작업 범위(task scope)가 얼마나 확장됐는지 평가한다

---

## 2. 상태 분류

| 상태 | 추정 사용량 | 판정 |
|------|-----------|------|
| 🟢 **양호** | < 40% | 계속 진행 가능 |
| 🟡 **주의** | 40–70% | 슬슬 정리 고려 |
| 🟠 **경고** | 70–85% | 새 세션 권고 |
| 🔴 **위험** | > 85% | 새 세션 강력 권고 |

---

## 3. 진단 리포트 출력

아래 형식으로 사용자에게 경고 메시지를 포함해 보고한다:

```
## 🔍 컨텍스트 상태 진단

**추정 사용량**: 약 X% (추정 ~N 토큰 / 200,000 토큰)
**상태**: [🟢 양호 / 🟡 주의 / 🟠 경고 / 🔴 위험]

### 감지된 신호
- [신호 목록 — 없으면 "특이사항 없음"]

### 파일/히스토리 요약
- 대화 turns: N회
- 로드된 파일: N개 (총 약 N줄)
- [반복 로드 파일 있으면 명시]

### ⚠️ 권장 조치
[상태에 따른 메시지 — 아래 액션 가이드 참고]
```

## 4. 액션 가이드

### 🟢 양호
> "컨텍스트 여유가 충분합니다. 계속 작업하세요."

### 🟡 주의
> "컨텍스트가 절반 이상 소모되었습니다. 큰 파일 재로드를 줄이고 불필요한 출력을 최소화하세요. 아직 새 세션이 필요하진 않습니다."

### 🟠 경고
> "⚠️ 컨텍스트가 상당히 소모되었습니다. 현재 작업 단위를 마무리한 뒤 새 세션 시작을 권장합니다."
>
> **새 세션 시작:**
> ```
> /new
> ```
> 새 세션 시작 전 현재 진행 상황을 메모해 두세요.

### 🔴 위험
> "🚨 컨텍스트가 한계에 근접했습니다. 지금 즉시 새 세션을 시작하세요. 이 상태에서 계속 작업하면 이전 내용이 누락되거나 오류가 발생할 수 있습니다."
>
> **즉시 새 세션 시작:**
> ```
> /new
> ```

## 참고: 새 세션 시작 전 체크리스트

새 세션(`/new`)을 시작하기 전에 다음을 확인하세요:

- [ ] 현재 작업 중인 파일 저장 여부
- [ ] 진행 중인 작업의 현재 상태 메모
- [ ] 다음 세션에서 필요한 컨텍스트 정리 (예: 어떤 파일, 어떤 목표)
- [ ] 미완성 변경사항 커밋 또는 스태시 여부
